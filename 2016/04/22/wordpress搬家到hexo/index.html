wordpress搬家到hexo

发表于 2016-04-22   |   分类于 瞎折腾   |     |   阅读次数   |  

前言

是的，看到这个界面的时候，说明我又换了一种风格，整个站点部署在github上，采用的框架是node.js编写hexo，至于如何在github上部署hexo，在这里就不再赘述了，google一下都是教程，按着步骤来做，应该不会出现太大问题的。在网上看到过这么一句话，说个人博客会经历三个阶段：

第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。


历程

的确，从最开始注册个人独立域名到现在的时间(http://tool.chinaz.com/DomainDel/?wd=alinwei.com)。差不多三年的时间，经历也是这几个阶段。从最开始的Emlog到大众的Wordpress，然后到现在的Hexo。空间也是经历这么几个阶段，最开始买的是一个200M的独立空间，那个时候由于很多东西不懂，导致买到的空间限制很多，最后到期后就没有继续续费。然后就是搬家到新浪的免费SAE，从最开始的免费到最近的收费，这之间大概经历了一年吧。收费幅度相比之前来说，跳跃性太大，实在是接受不了，刚好最近想折腾下，换一种风格，就打算重新捣鼓下。很早之前就接触过Hexo，那个时候只是感兴趣，想去折腾，最后部署在github上就没在去打理，然后就不了了之。最近重新折腾的时候，很多东西都忘记了，然后就是重新搭建本地将环境，加上最近工作非常繁忙，一直拖到现在才算完全部署到了github上了，接下来就是给优化和设置主题了。

hexo

Hexo的好处，谁用谁知道，网上有人这么来形容：
1、不可思议的快速 ─ 只要一眨眼静态文件即生成完成
2、支持 Markdown
3、仅需一道指令即可部署到 GitHub Pages 和 Heroku
4、已移植 Octopress 插件
5、高扩展性、自订性
6、兼容于 Windows, Mac & Linux

折腾

由于以前框架是采用wordpress，转到hexo虽说可以利用官方提供的hexo-migrator-wordpress插件来转换数据，但是由于里面的标签属性问题，大部分文章排版都是错位的，所以，最后要不要将那些数据放到这个上面来，还是待定，亦或者专门做个页面来存放这些个数据吧。
后面需要折腾的还有很多，比如说github毕竟是在国外，对于天朝来说，访问速度还是不敢苟同，不管你怎么去优化，还是达不到最理想的状态。所以，类似github这种代码托管的服务，国内还是有codeing（gitface），看到网上的教程说可以同时部署到github和gitface上，然后分别解析，国外直接访问github线路，国内访问gitface线路，这样速度就会大大提升。我想，这个就是后面需要折腾的吧。

结尾

我知道，坚持一件事很难，从最初的买域名、空间，折腾到现在2年8个月9天，记录的东西不多，但是也还是有一部分是自己内心的表达。所以，后面不管有多忙，希望自己还是坚持下去，保持最开始的那种心态。
好吧，就这，继续折腾中、 、 、
